# Expanding on the JIRA Skill

## Section Weights

Give each JIRA story section weights to prioritize information sources:

| Section                    | Weight | Notes                                                                                                                                                                                                                                                                                                                                                                                                    |
|----------------------------|--------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Title                      | 5      | Many times the title can't be trusted because it is a copy from another story                                                                                                                                                                                                                                                                                                                          |
| Description                | 8      | Okay for context but not great for backend changes                                                                                                                                                                                                                                                                                                                                                      |
| AC (Acceptance Criteria)   | 65     | Most of the time the AC is done well and kept up to date. That's what QA is going to focus on so that part must be implemented. Critical when Tech Details are missing                                                                                                                                                                                                                                                                           |
| Tech Details               | 100    | This section trumps the title and description. If missing, raise red flags                                                                                                                                                                                                                                                                                                                             |
| Comments                   | 30     | Sometimes the comments could hold helpful information from QA or devs. *Must ignore "Automation for Jira" bot messages*                                                                                                                                                                                                                                                                                              |
| Story Points               | 18     | The story points represent the complexity.<br>• 1 and 2 SP - simple change, little creativity<br>• 3 should be a clean implementation<br>• 5+ complex story that might need high creativity and more iterations. Break the PR into smaller PRs and ask for feedback from dev lead early in the process                                                                                                      |
| Story Status               | 10     | If in pre-refined status treat anything that's not the tech details as subject to change. Must review the Epic and Confluence docs                                                                                                                                                                                                                                                                        |

**Important:** If sections contradict each other, flag the contradiction and suggest corrections. When contradictions occur, prioritize based on the weight hierarchy (Tech Details > AC > Comments > Story Points > Description > Title), but always explicitly call out the conflict and recommend resolution.

## Epic Context Gathering

Use the `acli` tool to look up the story epic. From there, review the epic content for documentation that could be useful to build context for the story at hand. Collect all the stories under the epic and iterate over each to collect context. Note each story status (todo vs inprogress vs done) to build context. Look for open or merged PRs attached to each story and link to those so we can reference those when needed.
